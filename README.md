# Mediterra Fresh

Site vitrine B2B et playbook d'acquisition pour **Mediterra Fresh**, fournisseur
de fruits et légumes frais à Marrakech et Agadir.

> Built as a multilingual (FR / EN / AR with full RTL), SEO-optimised,
> mobile-first, sub-3s 4G load Next.js application — paired with a complete
> outreach playbook in `docs/`.

## Two systems, two domains — read this first

This repository contains **only System 1** (the public website + its inbound
contact form). The cold-outreach campaign is **System 2** — it lives on a
separate domain, uses different tools, and is documented as a playbook in
[`docs/`](./docs). They are deliberately kept apart so a cold-outreach mishap
cannot ruin the deliverability of your brand domain.

| | System 1 — this repo | System 2 — `docs/` playbook |
|---|---|---|
| Domain | `mediterra-fresh.com` (brand) | `mediterra-fresh.co` (sacrificial) |
| Direction | Inbound (you receive form submissions) | Outbound (you send to prospects) |
| Tool | Resend transactional API | Instantly / Smartlead + Google Workspace |
| Volume | ~10 emails/month | 30–60/day per inbox × 2–4 inboxes |
| Time to launch | 30 minutes | 3–4 weeks (mandatory warm-up) |

Full rationale and operating instructions: [`docs/00-architecture-two-systems.md`](./docs/00-architecture-two-systems.md).

## Stack

- **Next.js 15** (App Router) + **React 19** + **TypeScript 5**
- **Tailwind CSS 3** for styling — custom olive / terracotta / cream palette
- **next-intl** for FR / EN / AR with `localePrefix: 'always'` and `hreflang`
- **Resend** for the inbound contact form's transactional email. Free tier
  is enough for the early-stage volume — see [`docs/08-contact-form-resend.md`](./docs/08-contact-form-resend.md).
  Swap-out to Postmark / SES touches one file (`app/api/contact/route.ts`).
- **No external DB** — content lives in `lib/products.ts` and `messages/*.json`
- **No third-party tracking** by default. Privacy-friendly OpenStreetMap
  embed for the contact map (no Google API key required)

## Multilingual notes

- Default locale: `fr` (Morocco's primary business language)
- All routes are prefixed: `/fr/catalogue`, `/en/catalogue`, `/ar/catalogue`
- Arabic uses `dir="rtl"` on `<html>` and `Noto Naskh Arabic` font
- `hreflang` tags are emitted automatically by the layout
- Sitemap at `/sitemap.xml` lists all 3 language versions of every page

## SEO baked in

- LocalBusiness schema (JSON-LD) per page with hours, geo, areaServed
- `hreflang` for fr-MA / en / ar-MA + `x-default`
- Open Graph + Twitter card metadata per page
- `sitemap.xml` and `robots.txt` autogenerated
- Sub-3s LCP target on 4G (Tailwind purges, font swap, no client JS on
  static pages, only the language switcher and contact form ship JS)

## Deployment

The site runs on a self-hosted Ubuntu VPS behind Nginx, deployed by `git
pull`. Full step-by-step in [`docs/09-deploy-vps.md`](./docs/09-deploy-vps.md).

### Everyday workflow — push code, then deploy

```bash
# 1. Push to GitHub
git push origin main

# 2. Trigger a deploy on the server (pulls main, builds, reloads PM2)
ssh deploy@<your-server-ip> 'cd /var/www/trend-sites/mediterra-fresh.com && ./scripts/deploy.sh'
```

The deploy script (committed at `scripts/deploy.sh`) handles the whole
chain: `git fetch + reset --hard origin/main` → `npm ci` (only if the
lockfile changed) → `npm run build` → `pm2 reload mediterra-fresh` →
smoke-test `http://127.0.0.1:3000`. Total downtime per deploy: under 1
second (instant restart in fork mode).

If a deploy goes wrong:

```bash
ssh deploy@<your-server-ip> 'cd /var/www/trend-sites/mediterra-fresh.com && ./scripts/rollback.sh'
```

### One-time server setup

The first-time provisioning (Node 20, PM2, Nginx, UFW, Certbot, Cloudflare
Origin Cert) is documented end-to-end in
[`docs/09-deploy-vps.md`](./docs/09-deploy-vps.md). Run through it once on
a fresh VPS, then never touch it again — every subsequent deploy is the
two commands above.

### PM2 process model

[`ecosystem.config.cjs`](./ecosystem.config.cjs) at the repo root defines
the production app:

- `cwd: __dirname` — path-portable (works in `/var/www/trend-sites/...`
  *and* in `/srv/mediterra-fresh`).
- Bound to `0.0.0.0:3000` because Next.js's internal RSC fetcher resolves
  `localhost` via IPv6 first on dual-stack Linux. UFW (`22/80/443` only)
  keeps :3000 unreachable from the public internet — only Nginx on
  `127.0.0.1` talks to it.
- Logs at `/var/log/mediterra-fresh/{out,err}.log`.

### Pre-launch domain checklist

- [x] `mediterra-fresh.com` registered ✅ (brand domain — System 1)
- [ ] DNS A record for `mediterra-fresh.com` and `www.mediterra-fresh.com`
      points to the production server's public IP
- [ ] `mediterra-fresh.co` registered for the cold outreach domain (separate
      reputation — see `docs/03-dns-setup-spf-dkim-dmarc.md`)
- [ ] SSL active on the brand domain (Let's Encrypt via Certbot — automated
      in the deploy runbook)
- [ ] `siteConfig` updated in `lib/site.ts` with real numbers
- [ ] CNDP declaration filed and number entered in `siteConfig.legal.cndp`
- [ ] Real photos replace the gradient placeholders for at least the 20 hero SKUs
